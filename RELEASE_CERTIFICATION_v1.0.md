## Remediation Summary

Three HIGH issues were identified in `RELEASE_READINESS_REPORT.md` and resolved in a single remediation commit (`ede43b2`):

**H-2 (highest priority) — WB4 provenance gap:**  
`rta/trend_comparison_analysis.py` was reading Correction_Factor and n_eff from `ebc6aee6-Rainfall_2Trend_Results.xlsx` (WB4), which was a different 28-station dataset. The fix redirected the data source to Sheet S2 (`S2 Modified MK (H&R98)`) of the primary v4 Results workbook (WB1), computing CF = Var*(S)/Var(S) from committed data. Correct CF range: 1.0000–2.7251 (was: 1.0000–1.0984). Scientific results (Z-statistics, p-values, agreement rates) unchanged.

**H-1 — Core figures not version-controlled:**  
Created `results/archive_figures/` with 66 files (28 PNG + 28 PDF + 10 SVG) including Fig1–Fig8 which were previously excluded by the root `.gitignore`. Added SHA-256 checksums and figure manifest.

**H-3 — Manuscript placeholders:**  
Filled all 3 bracket placeholders in `Discussion_Template.md` and `Results_Template.md` with values traced to validated workbooks. Also corrected an inherited incorrect claim ("CF < 1.10 for all stations") to reflect the correct CF range.
